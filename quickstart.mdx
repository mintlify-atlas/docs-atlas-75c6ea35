This guide walks you through installing SNI-Spoofing, editing the configuration, and running the proxy so your application can bypass Deep Packet Inspection via a spoofed TLS SNI.

<Warning>
  SNI-Spoofing is **Windows-only**. It relies on WinDivert, a Windows kernel-level packet capture driver. It does not run on Linux or macOS.
</Warning>

## Prerequisites

- Windows 10 or later
- Python 3.10 or later
- Administrator privileges (required by WinDivert)
- `pip` available in your PATH

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/patterniha/SNI-Spoofing.git
    cd SNI-Spoofing
    ```
  </Step>
  <Step title="Install dependencies">
    SNI-Spoofing requires only one third-party package — `pydivert`, which provides Python bindings for WinDivert.

    ```bash
    pip install pydivert>=3.1.0
    ```

    WinDivert itself is installed automatically by pydivert the first time it loads.
  </Step>
  <Step title="Edit config.json">
    Open `config.json` in the project directory and set your target server and fake SNI:

    ```json config.json
    {
      "LISTEN_HOST": "0.0.0.0",
      "LISTEN_PORT": 40443,
      "CONNECT_IP": "188.114.98.0",
      "CONNECT_PORT": 443,
      "FAKE_SNI": "auth.vercel.com"
    }
    ```

    | Field | Description |
    |---|---|
    | `LISTEN_HOST` | IP to bind the local proxy (use `127.0.0.1` for local-only) |
    | `LISTEN_PORT` | Port your app connects to |
    | `CONNECT_IP` | IP of the remote server you want to reach |
    | `CONNECT_PORT` | Port on the remote server (typically `443`) |
    | `FAKE_SNI` | Hostname embedded in the fake TLS ClientHello |
  </Step>
  <Step title="Run as administrator">
    Open an elevated Command Prompt (right-click → **Run as administrator**), then:

    ```bash
    python main.py
    ```

    You should see the startup message in Persian (the author's support note) followed by the proxy listening.
  </Step>
  <Step title="Point your application at the proxy">
    Configure your application or tunnel client to connect to:

    ```
    127.0.0.1:40443
    ```

    (or your configured `LISTEN_HOST`:`LISTEN_PORT`). SNI-Spoofing will forward the traffic to `CONNECT_IP:CONNECT_PORT` with a fake TLS ClientHello injected.
  </Step>
</Steps>

## Choosing a FAKE_SNI

The `FAKE_SNI` value is embedded in a fake TLS ClientHello that is injected with an invalid TCP sequence number. DPI systems inspect this and see an allowed domain. The real server never processes this packet — it's discarded because the sequence number is out of window.

<Tip>
  Choose a domain that your DPI/firewall permits — for example, a major CDN or cloud provider hostname like `auth.vercel.com`, `www.google.com`, or `cloudflare.com`. The domain does not need to be reachable or related to your traffic.
</Tip>
