SNI-Spoofing is an open-source tool for bypassing Deep Packet Inspection (DPI) on Windows. It acts as a local TCP proxy that intercepts your outgoing connections and injects fake TLS handshake packets with a spoofed Server Name Indication (SNI) field — confusing DPI devices into believing traffic is destined for an allowed host while your real data flows unimpeded.

## What is SNI-Spoofing?

Many network censorship systems (DPI firewalls) inspect the TLS ClientHello's SNI field to decide whether to allow or block a connection. SNI-Spoofing defeats this by injecting a fake ClientHello with an innocuous SNI (such as `auth.vercel.com`) at an intentionally wrong TCP sequence number. The DPI device reads and evaluates the fake packet, while the actual TLS handshake — arriving at the correct sequence — is accepted by the real server and ignored by the DPI.

## Getting started

<Steps>
  <Step title="Install Python and WinDivert">
    SNI-Spoofing requires Python 3.10+ on Windows and the `pydivert` package, which wraps the WinDivert kernel driver for packet interception.
  </Step>
  <Step title="Install dependencies">
    ```bash
    pip install pydivert>=3.1.0
    ```
  </Step>
  <Step title="Edit config.json">
    Set `CONNECT_IP` to your target server, `CONNECT_PORT` to its port (typically 443), and `FAKE_SNI` to a publicly accessible hostname your DPI allows.
  </Step>
  <Step title="Run the proxy">
    ```bash
    python main.py
    ```
    Point your application at `127.0.0.1:40443` (or your configured `LISTEN_PORT`).
  </Step>
</Steps>

<Note>
  SNI-Spoofing must be run with administrator privileges on Windows so that WinDivert can install its kernel-level packet capture driver.
</Note>
